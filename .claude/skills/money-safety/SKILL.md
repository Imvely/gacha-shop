# 돈/재고 코드 안전 게이트

이 프로젝트에서 돈과 재고는 사업의 생명선이자 법적 리스크다. 아래를 어기는 코드는
어떤 이유로도 커밋하지 않는다.

## 설계 불변식 (schema.sql과 일치해야 함)

1. **잔액은 컬럼이 아니라 원장이다.** 잔액 갱신 = `wallet_ledger`에 행 추가.
   `users`에 balance 컬럼을 만들거나 UPDATE로 잔액을 고치는 코드는 금지.
   모든 원장 행은 `reason` + `ref_type`/`ref_id`로 원인 객체와 연결한다.
2. **추첨 + 재고 차감 + 원장 차감 + 감사 로그 = 단일 DB 트랜잭션.**
   `machine_items` 행은 `SELECT ... FOR UPDATE`로 잠근 뒤 읽는다.
   트랜잭션 밖에서 stock을 읽고 안에서 차감하는 TOCTOU 패턴 금지.
3. **감사 로그 필수.** 모든 draw에 rng_value, stock_snapshot(JSONB), seed 커밋-리빌 체인.
4. **클라이언트 값 불신.** 금액·수량·가격은 항상 서버에서 재계산. 요청 body의
   금액을 그대로 원장에 쓰는 코드 금지.
5. **결제 확정은 웹훅만.** 프론트 결제 성공 콜백으로 `payments.status='paid'` 전환 금지.
   `pg_tx_id` UNIQUE로 웹훅 중복 수신(멱등성)을 방어한다.
6. **현금 환급 경로 자체를 만들지 않는다.** 교환소 보상은 코인 재화로만.
