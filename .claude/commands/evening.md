# /evening — 하루 마감 루틴

오늘 세션을 마감한다. 아래 순서를 빠짐없이 수행하고, 마지막에 마감 보고를 남겨라.

## 컨텍스트 (자동 주입)

- 오늘 날짜: !`date +%Y-%m-%d`
- 워킹트리 상태: !`git status -sb`
- 오늘 커밋: !`git log --oneline --since="6am" 2>/dev/null || echo "(no commits today)"`
- 기능 백로그: @features.json

## 수행 순서

1. **오늘 회고**: 이 세션에서 실제로 한 일을 정리하라. git log·변경 파일과 대조해서
   "실제로 완료된 것"과 "하다 만 것"을 구분하라. 과장 금지 — 테스트가 안 돌았으면 미완료다.

2. **워크로그 작성**: `docs/worklog/YYYY-MM-DD.md` (오늘 날짜)를
   `docs/worklog/TEMPLATE.md` 형식으로 작성하라. 이미 있으면 덮지 말고 병합/추가하라.
   - 오늘 한 일 (커밋 해시 참조)
   - 검증 결과 (돌린 테스트와 실제 결과)
   - 결정 사항 / 문제·미해결 / 내일 할 일

3. **features.json 갱신**: 오늘 상태가 바뀐 기능의 status를 갱신하라.
   단, `done` 전환은 /done 게이트(.claude/commands/done.md)를 통과한 기능만.
   오늘 /done을 안 거쳤다면 여기서 몰래 done으로 바꾸지 마라 — in_progress로 남겨라.

4. **STATE.md 갱신**: "진행 중인 기능", "다음에 할 일"(내일 기준 우선순위순),
   "막힌 것", "마지막 세션 요약"을 오늘 기준으로 다시 써라. 히스토리는 남기지 말 것
   — STATE.md는 항상 현재 스냅샷만 유지한다.

5. **커밋 & 푸쉬**:
   - 변경사항을 논리 단위로 커밋하라. 기능 작업 커밋 메시지에는 `[F-xx]` 포함.
   - 마감 커밋 메시지 예: `chore: 2026-07-17 worklog & state update`
   - `git push`로 원격에 반영하라. 원격(origin)이 없으면 푸쉬는 건너뛰고
     마감 보고에 "원격 미설정 — 푸쉬 생략"이라고 명시하라.

6. **메모리 저장**: 오늘 알게 된 것 중 코드/로그로는 남지 않는 지속 가치가 있는 것만
   메모리 디렉토리에 저장하라 (사용자 선호, 프로젝트 방향 결정, 반복 실수 교훈 등).
   저장할 게 없으면 억지로 만들지 마라.

7. **마감 보고 (한마디)**: 다음 형식으로 끝내라.
   > 🌙 오늘: (완료한 것 요약) / 커밋: N개 (푸쉬 여부) / 내일: (1~3개) / 메모: (특이사항)
