# /morning — 하루 시작 루틴

너는 이 프로젝트(PONG! 온라인 가챠샵)의 단독 개발 파트너다. 사용자가 지켜보지 않아도
스스로 판단해서 하루 작업을 시작해야 한다. 아래 순서를 그대로 수행하라.

## 컨텍스트 (자동 주입)

- 오늘 날짜: !`date +%Y-%m-%d`
- 현재 브랜치/상태: !`git status -sb`
- 최근 커밋 15개: !`git log --oneline -15 2>/dev/null || echo "(no commits yet)"`
- 프로젝트 상태 파일: @STATE.md
- 기능 백로그: @features.json

## 수행 순서

1. **어제 파악**: `docs/worklog/`에서 가장 최근 로그 파일을 찾아 읽어라(오늘 날짜 제외).
   위 git log와 대조해서 어제(또는 마지막 세션) 실제로 무엇이 완료됐는지 확인하라.
   로그에 "했다"고 적혀 있어도 커밋/파일이 없으면 미완료로 간주한다.

2. **아침 브리핑 (한마디)**: 다음 형식으로 짧게 보고하라.
   > ☀️ 어제: (1~2문장 요약) / 오늘: (오늘 할 일 1~3개) / 리스크: (있으면)

3. **오늘 할 일 결정**: 우선순위 규칙 —
   - STATE.md의 "다음에 할 일"과 "막힌 것"이 1순위
   - 그다음 features.json에서 `in_progress`인 기능 (마무리 우선)
   - 없으면 phase P1의 `todo` 중 의존성 순서상 가장 앞선 것
   - 작업 시작 시 해당 기능의 status를 `in_progress`로 갱신하라 (CLAUDE.md 8절)

4. **작업 계획 등록**: TaskCreate로 오늘의 작업을 3~6개 단위로 등록하라.

5. **자동 진행**: 승인을 기다리지 말고 바로 첫 번째 작업을 시작하라.
   단, 다음은 예외적으로 멈추고 물어본다: 되돌리기 어려운 파괴적 작업,
   결제·배포 등 외부 서비스에 실제 영향을 주는 작업, features.json 범위 밖의 큰 방향 전환.

6. **작업 중 규칙**: CLAUDE.md의 절대 원칙(서버 추첨, 단일 트랜잭션, 감사 로그,
   현금 환급 금지, 실데이터 확률, 서버 재계산)을 항상 지켜라.
   돈/재고 코드를 만졌다면 동시성 테스트(스레드 20개) 통과 전에는 done 처리 금지.
